# 1️⃣ Color_Timer_Spread

![alt text](Screenshots/candle.png)

![alt text](Screenshots/Timer.jpg)

![alt text](<Screenshots/Line Change.jpg>)

## Purpose

A **real‑time chart monitoring indicator** that provides key trading
metrics directly on the chart interface.

It helps traders monitor:

- Time remaining for current candle
- Can also set custom lead alert for candle closing
- Shows Current spread
- Bid/Ask visualization
- Optimises UI based on time frame
- Adv functionality to manupulate specific chart objects in all open charts
- Shows Color pallate

## Key Features

### Candle Timer

Displays the **remaining time before the current candle closes**.

This helps traders:

- Avoid entering trades at the end of candles
- Time entries more precisely

### Spread Monitor

Shows **real‑time spread values**.

Useful for:

- Avoiding trades during high spread
- Monitoring broker execution conditions

### Bid / Ask Visualization

Optionally displays **Bid and Ask price lines** on the chart.

This allows traders to:

- See execution prices clearly
- Understand spread visually

### Visual Interface Elements

Uses chart UI elements such as:

- Labels
- Buttons
- Color‑coded display panels

to create a compact **trading dashboard inside MT5**.

# 2️⃣ EventLines

![alt text](Screenshots/events.jpg)

![alt text](Screenshots/Event-lines.jpg)

## Purpose

This indicator **automatically draws vertical time‑based event lines**
on the chart.

It helps traders visualize important market session timings and trading
windows.

## Key Features

### Trading Session Markers

Automatically draws vertical lines for:

- Trading start
- London session start
- US session start
- Key market move times
- Trading end

### Color‑Coded Market Events

Each event line has its own color and tooltip.

Example events:

- London session open
- US session open
- Key volatility windows

### Configurable Visibility

Each event type can be **enabled or disabled individually**.

This allows traders to customize their chart for:

- Intraday trading
- Session‑based strategies

### External Data Support

The indicator attempts to **read event timing data from a file**.

Generated by "`Raw-Data-Processors_JS`".

If the file is missing, it falls back to **default trading session
times**.

This demonstrates:

- File handling
- Fault‑tolerant design
- Configuration‑driven behavior

# 3️⃣ Pips_Counter

![alt text](Screenshots/Pips_Counter.jpg)

## Purpose

A **performance tracking indicator** that monitors trading performance
directly on the chart.

It calculates:

- Daily profit/loss
- Weekly profit/loss
- Number of trades taken

This creates a **live trading performance dashboard**.

## Key Features

### Daily Performance Tracking

Displays:

- Total pips gained/lost today
- Total money gained/lost today

Updated automatically when new trades close.

### Weekly Performance Tracking

Aggregates:

- Weekly pips
- Weekly profit

This allows traders to monitor **larger performance trends**.

### Trade Counter

Tracks how many trades have been taken during the session.

This helps enforce **trading discipline rules** such as:

- Maximum trades per day

### Automated Guardrails

The indicator can trigger **external automation scripts** when certain
limits are reached.

Example:

- Launch reminder programs after specific trade counts
- Automatically stop trading after loss thresholds

This demonstrates:

- System integration
- Process automation

### Visual Dashboard

All statistics are displayed using chart labels such as:

    Weekly: 120 pips | $340
    Daily:  25 pips | $70
    Trades Taken: 3

# 4️⃣ Trade‑Alert

![alt text](Screenshots/Alerts.jpg)

## Purpose

An **intelligent price alert system** that monitors chart levels and
alerts when price approaches important zones.

This tool acts as a **real‑time monitoring system for trading levels**.

## Key Features

### Support / Resistance Detection

The indicator monitors:

- Horizontal lines
- Rectangle zones

These typically represent:

- Support
- Resistance
- Supply zones
- Demand zones

### Proximity Alerts

Alerts trigger when price approaches a level within a configurable
sensitivity range.

Example:

    sensitivity = 0.00002

This allows early warnings before price reaches key levels.

### Candle Confirmation Logic

On higher timeframes, alerts only trigger when:

- Candle structure confirms the move
- Valid bullish or bearish candles appear

This reduces **false alerts**.

### One‑Time Alerts

Each level triggers alerts **only once**, preventing spam notifications.

### Market Direction Validation

If price invalidates a zone:

- The zone fades
- The system alerts the trader

This helps traders quickly recognize **invalid setups**.

### Interactive Alert Button

The indicator includes a **chart button** to enable or disable alerts
directly from the interface.
